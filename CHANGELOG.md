# Changelog

このファイルは AI Code Guide のバージョンごとの変更点を記録します。
バージョンは [Semantic Versioning](https://semver.org/lang/ja/) に従います（新機能=minor / バグ修正=patch）。

配布は研究室内向けに GitHub Releases で行っています。更新方法は各リリースの手順に従ってください（自動更新はされないため、新しい vsix を手動で入れ直します）。

## [0.19.2] - 2026-07-21

### 変更
- 下線（トークン）の解説をホバーで出したとき、解説の先頭に対象のコード1行を表示するようにしました。ホバーの吹き出しがコードに重なっても、何についての解説かが吹き出しの中だけで分かります

## [0.19.1] - 2026-07-21

### 修正
- 解説の生成が「解説を生成できませんでした」で終わってしまうことがある問題を軽減しました。AIの返答が途中で壊れていても、正しく届いている分だけを表示します（従来は1箇所でも壊れていると全部捨てていました）
- 上記が起きたとき、AIの返答そのものを「表示 > 出力 > AI Code Guide」に残すようにしました。原因が分かるようになります
- 解説が0件だったときのメッセージに、次に何を見ればよいか（出力パネル）を書くようにしました

## [0.19.0] - 2026-07-21

### 追加
- 下線（トークン）の解説の出し方に「ホバー」を追加。設定「下線の解説の位置」または解説タブの切り替えで、`行の下` / `右余白` / `ホバー` の3つから選べます。`ホバー` にすると点線下線だけが常に見え、解説はマウスを乗せたときに出ます（コードの行間が空かず、複数行を続けて読みたいときに向きます）。既定は従来どおり `行の下` です

### 変更
- 下線にホバーしたときの文を「それだけ読んで分かる説明」に作り直しました。従来は短い見出しと補足の2段だったため、見出しを見ずに補足だけ読むと話が途中から始まっていました。今後は1文目で「これが何か」、続けて「なぜこう書くか・注意点」を述べます（ブロック枠の解説は従来どおり）
- この変更に伴い、保存済みの解説は作り直しになります（同じファイルでももう一度生成されます）
- 重なったブロック解説の切り替えを、コード行の上に出る行から**ブロックのホバー内のリンク**へ移しました（「重なった説明 1/3 ・ 次を表示」）。コードの行間に余計な行が入らなくなります

### 修正
- 重なったブロックがある場所でコード本体にホバーすると、画面に表示されている解説ではなく、後ろに隠れている別の解説が出ることがあった問題を修正
- 下線の解説を「右余白」または「ホバー」にしているとき、ホバー内に「変更は行末のチップから」と案内が出るのに、そのチップが存在しなかった問題を修正（案内を出さないようにしました）

## [0.13.0] - 2026-07-16

### 追加
- **設計ファイル機能（フェーズ1）を追加しました**: コードとは別に「このコードは何のために・どういう方針で書かれたか」を記録する設計ファイル（`.ai-code-guide/design/` 配下のMarkdown）を読み込み、各タブに表示します。
  - **作る**: コマンド「AI Code Guide: 設計ファイル作成プロンプトをコピー」または各タブに出る「設計ファイルを作る」ボタンで、依頼プロンプトをコピーできます。Claude Code / codex に貼ると、あなたへのヒアリングを経て設計ファイルが作られます（拡張自体はAI生成しません。開発者が確認した内容 `確認済` と、AIがコードから推測した内容 `推測` をタグで区別して記録します）。
  - **見る**: プロジェクトタブ=リポジトリ全体・ディレクトリの設計とファイルカードの「設計」バッジ / 概要タブ=ファイルの目的 / 標準タブ=関数・クラスカードに目的・要件・制約・方針・構成の「設計」ブロック。すべての記述に出所バッジ（確認済=緑 / 推測=グレー / 未確認=橙）が付きます。
  - **検証**: 設計ファイルの書式違反や、コードに見つからない参照は警告として該当箇所に表示します。
  - 設計ファイルを編集すると、開いているタブへ自動で反映されます。

## [0.11.0] - 2026-07-08

### 追加
- **チャットでコードを引用して質問すると、その箇所にAIの結論が解説として残るようになりました**: エディタでコードを選択して引用付きで質問すると、応答のあとにその引用箇所へ自動で解説が付きます。1行だけの引用は点線の下線、複数行にまたがる引用は枠で囲んで示します。表示される見出しは「会話の結論（＝この箇所が何をしているか）」で、会話を続けるたびに最新の結論へ更新されます。
  - 下線には短い一言の解説、枠にはやや詳しい解説と、囲みの大きさに合わせて長さを出し分けます。
  - 見出しにマウスを乗せると、より詳しい補足と「💬 会話へ」リンクが出て、元のチャットに戻れます。
  - コードを編集すると位置は自動で追従し、ファイルを開き直しても解説は復元されます。

## [0.8.0] - 2026-07-06

### 追加
- **チャットの引用が文中に入るようになりました**: エディタでコードを選択してコピーし、チャットの入力欄に貼り付けると、生のコードの代わりに**引用チップ（`引用1 ファイル名 L83 「コード先頭」`のような小さなタグ）がカーソル位置に**入ります。チップの前後にそのまま文章を書けるので、「こっちのやつ(引用1)はいいけど、こっちのやつ(引用2)は直して」のように**複数の引用を文中で指し分けて**質問できます（AIにも番号付きで正しく伝わります）。チップは×ボタンでもBackspaceでも外せて、残った引用の番号は自動で振り直されます。送信後の自分の吹き出しでも、引用は文中のその位置にチップとして表示されます。
- **AIの返信中でも次のメッセージを送れるようになりました**: 返信待ちの間も入力・送信ができ、送った分は「あなた（送信待ち）」として薄く表示され、返信が終わると自動で順番に送信されます。返信が来た瞬間に入力途中の文章が消えることもなくなりました。
- **カードの▶展開に「概要」が付きました**: 関数カードを▶で展開すると、サブカード（処理の分解）の前に**目的・入力・出力（・補足）**の説明が色付きタグで表示されます。「この関数が何のためにあるか」「何を受け取って何を返すか」が先に分かるので、上から読むときの前提が掴めます（補足には「先に既知IDを取得しているのは重複チェックのため」のような非自明な背景が入ります）。

### 変更
- **チャットの吹き出しを色分けしました**: 自分=青系、AI=グレー系で見分けやすくなりました。
- **モジュールマップ（標準タブ）のレイアウトを刷新しました**: カードがサイドバーの端まで使う全幅表示になりました。▶展開したグループは「メインカード=濃い親色 / 概要・サブカード=薄い親色 / カードの継ぎ目と下端=親色の帯」という同じ色の濃淡で表示され、どのカードがどの関数の配下かが色で追えます。サブカードの左インデントは廃止し、各カードには細い枠線で輪郭を付けています。

### 修正
- **インライン説明の「↑」の位置ズレを直しました**: コード下の説明（↑付きのCodeLens）の矢印が、指すべきトークンの下線から右や左に数文字ズレることがありました。位置計算を実測に基づいて補正し、下線の真下を指すようになりました。

## [0.7.2] - 2026-07-06

### 修正
- **▼展開サブブロックの背景色が文の途中で切れなくなりました**: モジュールマップ（標準タブ）でカードを▼展開したとき、サブブロックの色分けが複数行の文（複数行にわたる関数呼び出しなど）の途中で切れたり、どの色にも塗られない行が残ったりすることがありました。AIが提案した行の区切りをそのまま使わず、**Python の文法上の文の区切りに機械的に合わせ直す検証層**を追加したため、色の境界は必ず文の切れ目になり、隙間もなくなります。あわせてラベルがブロック内の一部の処理だけを名乗らないよう生成ルールを改善しました（既存の展開結果は作り直されます）。

## [0.7.1] - 2026-07-04

### 修正
- **カードのAI説明が無駄に作り直されなくなりました**: モジュールマップ（標準・概要タブの関数／クラスのカード）でAI説明を付けたあと、ファイルのどこか一箇所を直して「AI説明」を押し直すと、これまでは**関係のないカードまで全部**作り直され（＝そのぶんトークンを消費し）ていました。今回から、**中身が変わったカードだけ**が作り直され、変えていないカードは前の説明をそのまま再利用します。ファイルの先頭に関数を足すなど、位置が大きくずれる編集でも同じく再利用されます（インライン説明が部分編集に強いのと同じ考え方に揃えました）。上部のファイル全体の見出しも、関数の中身をいじっただけでは作り直されず、関数・クラスの構成が変わったときだけ更新されます。

## [0.7.0] - 2026-07-04

### 追加
- **フローチャートが編集に自動追従**: 標準フローチャート（モジュールマップ）を表示したまま、そのファイルを編集すると、**保存やタブの切り替えをしなくても**約0.5秒後に自動で最新の内容へ描き直されるようになりました。自分でコードを打ったときも、Claude Code など外部ツールがファイルを書き換えたときも同じように追従します。これまでは編集してもフローチャートが古いまま残り、行の背景色ハイライトがずれることがありましたが、その状態が解消されます（AIの再生成は走らないためトークンは消費しません）。

### 追加
- **チャットのモデル表示・切替**: チャット入力欄のすぐ上に、いま会話に使っているモデル（例: `Claude Sonnet（サブスク）`）を常に表示します。バッジをクリックすると切り替えられます。**選べるのは今の認証で実際に使えるモデルだけ**です（Claude サブスク利用中なら Claude のみ、API キーを登録したプロバイダのモデルだけが並びます。使えない GPT/Gemini などは表示されません）。
- **チャットの「思考の深さ（effort）」切替**: モデルの隣のバッジで、AIがどれだけ深く考えるかを `low / medium / high` から選べます。**effort（エフォート）＝ 思考にかける手間**のことで、深いほど賢い代わりに遅く・コストが高くなります。既定は `low`（従来どおり高速）。表示名は選んだモデルに合わせて変わります（Claude=`effort` / GPT=`reasoning` / Gemini=`thinking`）。プロバイダごとに指定方法が違いますが、アプリ側で吸収します。effort に対応しないモデル（Claude Haiku 4.5 など）では無視されます。
- **インライン説明のキャッシュ自動復元**: 一度AI説明を生成したファイルは、閉じて開き直すと（自動生成をOFFにしていても）保存済みの説明がすぐ再表示されます。トークンは消費しません。

### 変更
- **引用チップを入力欄の中へ（Cursor風）**: 引用したコードの場所参照（例: `algorithms.py L19-24`）を、入力欄の中の上部にコンパクトなチップとして表示するようにしました。複数の箇所を同時に引用でき、それぞれ `×` で個別に外せます。従来の「引用」ラベルや「コードを表示」トグルは省き、場所だけをすっきり示します。
